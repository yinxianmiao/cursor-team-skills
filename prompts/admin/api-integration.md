# 接口联调（通用）

> 我只给 `doc/api/*.md`，不另给接口白名单。联调前必读本文；填空模板见 `docs/prompts.md`「接口联调」。
> 本文优先于通用 skill `vue3-admin-page-api`；冲突以本文为准。

## 执行约定

- 基于已有页面做接口联调，直接改代码；不要扩展需求，不要做无关重构
- **栈**（用户模板不填）：先自动识别 **若依 Vue3** / **PIGX** / **自研**（对照 `docs/prompts/conventions.md` 栈表与仓库特征）；列表/分页字段以 doc 及同模块、同栈参考页为准
- **硬约束**：保留页面表单项/文案/布局；仅改联调请求与赋值；doc 缺字段或对不上时先问，禁止擅自删项
- **联调后删除 mock 数据**（页面/组件内占位列表、假接口、本地 mock 文件中本任务相关部分）
- 仅改任务列出的文件；多改先问

## 前置

1. 识别栈：**若依 Vue3** / **PIGX** / **自研**（见 `docs/prompts/conventions.md` 栈表；勿混用列表模式）
2. @ `vue3/api-js.mdc` + `admin/api.mdc` + `admin/pages.mdc`；范围冲突取更严。

## 接口来源（强制）

- **唯一依据**：本次任务列出的 `doc/api/*.md`（仅以各文件内接口清单为准）
- **禁止**：调用 md 未出现的 URL；import/改动 md 未涉及的其它 `src/api/**`；查阅其它 md、全站 grep、相似页面擅自补接口
- **md 未写**：树/下拉/字典等 → 静态或现有 mock，不得换其它模块真接口
- **md 只写筛选参数、未写数据源**（如区划树）：点击可把 query 带给 md 分页接口，**不要**为树单独请求
- **页面旧代码**含文档外 import/请求 → 去掉或恢复占位，不因「原来就有」保留
- **md 不够用** → 先问我，不擅自扩需求

## 联调任务

1. 只读上述 md 的 method、path、入参、出参后再动手
2. mock/占位换为 md 内 HTTP 接口；`src/api/【模块】/` 仅封装 md 内接口
3. **请求/响应字段**与 md 一致（入参、出参、枚举）；**表单项/文案/布局**一律保留，禁止按 md「裁剪」页面字段
4. 保留 UI/交互，只改联调必需代码；md 缺字段或与页面字段对不上时先问，不擅自删项、不扩需求
5. `components/` 内弹窗/表单优先在子组件改接口

## 写/改 API

- 采用项目既有 request 封装（跟同模块 API 文件写法一致）
- GET：`params`；POST/PUT：`data`
- 路径参数仅契约明确要求时用 `` `/xxx/${id}` ``
- 页面只 import API 方法，不改方法名（除非我授权）

## 页面接入

- 直接赋值，避免无依据 `map`
- 错误交给 `utils/request` 全局拦截，页面默认不重复 catch/toast
- 分页字段以 doc + 同模块参考页为准（若依常见 `rows`/`total`；PIGX 常见 `data.records`/`data.total`）

## 联调排查

| 现象 | 查什么 |
|------|--------|
| 404 | API `url` vs doc；`.env` 代理配置 |
| 401 | token；request 拦截器 |
| 参数未生效 | Network 看实际 query/body；字段名是否与 doc 一致 |
| 有响应无数据 | 响应层级；表格 `prop` 与字段名 |
| 分页异常 | 请求分页字段与 `total` 来源是否与后端一致 |

验证顺序：Network → 列表 → 详情 → 提交 → 空数据。

## 规范

- `vue3/base.mdc` + `vue3/api-js.mdc` + `admin/api.mdc`
- `async/await`；数据模型 → 生命周期 → 业务方法
- 未经我授权：不改已有 params/data 字段名、不写多别名兜底
